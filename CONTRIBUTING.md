# Contributing to Red9inja-GPT

Thank you for your interest in contributing to Red9inja-GPT!

## How to Contribute

1. **Fork the repository**
2. **Create a new branch** (`git checkout -b feature/amazing-feature`)
3. **Make your changes**
4. **Test your changes**
5. **Commit** (`git commit -m 'Add amazing feature'`)
6. **Push** (`git push origin feature/amazing-feature`)
7. **Open a Pull Request**

## Development Setup

```bash
# Clone your fork
git clone https://github.com/yourusername/red9inja-GPT.git
cd red9inja-GPT

# Install dependencies
pip install -r requirements.txt

# Run tests
python quickstart.py
```

## Code Style

- Follow PEP 8 guidelines
- Use type hints where possible
- Add docstrings to functions and classes
- Keep functions focused and modular

## Testing

Before submitting a PR:
- Test your code thoroughly
- Ensure all existing tests pass
- Add new tests for new features

## Areas for Contribution

- **Model improvements**: New attention mechanisms, optimizations
- **Training**: Better training strategies, data augmentation
- **Inference**: Faster generation, better sampling methods
- **Documentation**: Tutorials, examples, explanations
- **Tools**: Visualization, analysis, debugging tools
- **Datasets**: Data preprocessing, tokenization improvements

## Questions?

Open an issue for any questions or discussions!
